# vJailbreak

vJailbreak is a VMware to Platform9 Private Cloud Director(PCD) VM migration tool.

It is built on Kubernetes (k3s). It converts VMDK disks to QCOW2 (or raw LUNs) and deploys VMs to OpenStack, supporting hot (live) and cold migrations with Changed Block Tracking (CBT).

For deep architecture context, CRD definitions, controller descriptions, migration state machine, log patterns, and debugging commands, see [DEVELOPER_DOCUMENTATION.md](DEVELOPER_DOCUMENTATION.md).

Use https://deepwiki.com/platform9/vjailbreak for more information.

## Repository Layout

| Path | Purpose |
|------|---------|
| `k8s/migration/` | Kubernetes controller manager (Go, controller-runtime) |
| `v2v-helper/` | Migration worker pod — disk copy and conversion (Go, libguestfs) |
| `ui/` | React/TypeScript frontend (MUI, Vite) |
| `pkg/vpwned/` | REST API server (Go) for Cluster Conversion |
| `pkg/common/` | Shared Go utilities |
| `image_builder/` | Builds the vJailbreak appliance QCOW2 image |
| `appliance/` | Vagrant-based k3s cluster for local testing |
| `deploy/` | Generated Kubernetes manifests |
| `docs/` | Astro documentation site |
| `scripts/` | Utility and firstboot scripts |

## Build Commands

```bash
# One-time setup
make setup-hooks

# Build individual components (Docker images)
make ui                    # UI image
make v2v-helper            # v2v-helper image
make vjail-controller-only # Controller image (no v2v rebuild)
make vjail-controller      # Controller + v2v-helper images
make build-vpwned          # API server image

# Generate deploy/installer.yaml
make generate-manifests

# Build complete appliance QCOW2
make build-image

# Lint
make lint

# Run controller locally (requires KUBECONFIG)
make run-local
```

Image tags default to `<git-parent-branch>-<short-sha>`. Override with:
```bash
BUILD_VERSION=v1.2.3 make <target>
REGISTRY=myregistry.io REPO=myorg make <target>
```

## Key CRDs

All live in namespace `migration-system` (or as configured).

| CRD | Purpose |
|-----|---------|
| `Migration` | Tracks one VM migration (primary workload object) |
| `MigrationPlan` | Groups multiple migrations |
| `VMwareCreds` | vCenter credentials |
| `OpenstackCreds` | OpenStack credentials |
| `NetworkMapping` | VMware portgroup → OpenStack network |
| `StorageMapping` | VMware datastore → OpenStack volume type |
| `MigrationTemplate` | Reusable migration config |

## Important Conventions

- **CRD changes**: After editing types in `k8s/migration/api/v1alpha1/`, run `make generate` inside `k8s/migration/` to regenerate deepcopy/client code and update CRD YAML.
- **installer.yaml**: Do not hand-edit `deploy/installer.yaml` — it is generated by `make generate-manifests`.
- **v2v-helper tests**: CGO_ENABLED=1 is required; tests will not compile on macOS without a Linux cross-compilation toolchain.
- **Git hooks**: `make setup-hooks` must be run once per clone to activate pre-commit validation.
