# Contracts: Extended `MigrationTemplate` CRD & REST paths

## CRD: `MigrationTemplate` (modified)

`apiVersion: vjailbreak.k8s.pf9.io/v1alpha1`, `kind: MigrationTemplate` — existing kind, extended schema.

### Example: saved template (post-feature)

```yaml
apiVersion: vjailbreak.k8s.pf9.io/v1alpha1
kind: MigrationTemplate
metadata:
  name: production-rhel-east
  namespace: migration-system
  labels:
    vjailbreak.k8s.pf9.io/saved: "true"
spec:
  saved: true
  displayName: "Production RHEL · East"
  description: "Standard hot migration for east-region RHEL web & app tiers. Admin-gated cutover, Ceph NVMe storage."
  osFamily: linuxGuest
  networkMapping: netmap-east-prod
  storageMapping: stormap-east-prod
  storageCopyMethod: normal
  source:
    vmwareRef: vcenter-east-creds
  destination:
    openstackRef: pcd-east-1-creds
  targetPCDClusterName: cluster-prod-a
status:
  timesUsed: 28
  lastUsedAt: "2026-06-22T00:00:00Z"
```

### Example: ephemeral per-session template (existing behavior, unchanged)

```yaml
apiVersion: vjailbreak.k8s.pf9.io/v1alpha1
kind: MigrationTemplate
metadata:
  name: 3fa2c1de-9b7a-4e21-8c31-9a2f6b9d4c11   # uuid, as generated today
  namespace: migration-system
spec:
  # saved omitted/false — never shown in Templates tab, remains subject to
  # useCredentialFetching.ts auto-patch and useMigrationFormSubmit.ts auto-delete-on-cancel
  osFamily: windowsGuest
  source:
    vmwareRef: some-creds
  destination:
    openstackRef: some-other-creds
# status omitted — never written for ephemeral templates
```

### Full extended `MigrationTemplateSpec` schema (additive fields only; existing fields unchanged — see current `migrationtemplate_types.go`)

```yaml
properties:
  spec:
    properties:
      displayName:
        type: string
      description:
        type: string
      saved:
        type: boolean
  status:
    properties:
      timesUsed:
        type: integer
      lastUsedAt:
        type: string
        format: date-time
```

Regenerated by `make generate` (kubebuilder markers in `migrationtemplate_types.go`) — this block is illustrative, not hand-maintained; the authoritative schema lives in `k8s/migration/config/crd/bases/vjailbreak.k8s.pf9.io_migrationtemplates.yaml` post-generation.

## REST API (existing generic Kubernetes custom-resource path — no new backend service)

Base path (per `ui/src/api/constants.ts`): `/apis/vjailbreak.k8s.pf9.io/v1alpha1/namespaces/{namespace}/migrationtemplates`

| Operation | Method + Path | Notes |
|---|---|---|
| List saved templates | `GET .../migrationtemplates?labelSelector=vjailbreak.k8s.pf9.io%2Fsaved%3Dtrue` | New — filters out ephemeral per-session objects |
| Get one template | `GET .../migrationtemplates/{name}` | Existing, unchanged |
| Create saved template | `POST .../migrationtemplates` | Body includes `spec.saved: true`, `spec.displayName`, `spec.description`, plus existing mapping/source/destination fields; `metadata.name` = sanitized `displayName`; `metadata.labels["vjailbreak.k8s.pf9.io/saved"] = "true"` set alongside |
| Clone template | `POST .../migrationtemplates` | Same as Create, with `spec` copied from the source template and a new unique `metadata.name`/`spec.displayName` |
| Delete template | `DELETE .../migrationtemplates/{name}` | Existing, unchanged |
| Update usage stats | `PATCH .../migrationtemplates/{name}/status` (`Content-Type: application/merge-patch+json`, body `{"status":{"timesUsed": N, "lastUsedAt": "<ISO-8601>"}}`) | New — status-subresource PATCH; `ui-manager-role` already grants `migrationtemplates/status` get/patch/update, no RBAC change needed |

No changes to `pkg/vpwned/`'s REST surface — all of the above goes through the existing nginx/vite-proxied path straight to the Kubernetes API server, exactly as today's ephemeral-template CRUD already does.
