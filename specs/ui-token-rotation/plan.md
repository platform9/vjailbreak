# Implementation Plan: UI ServiceAccount Token Security

> **Note**: Task IDs in this plan (T001–T030) are **design-phase references only** and do not match `tasks.md` implementation IDs (T001–T021). Use `tasks.md` as the authoritative execution sequence.

**Branch**: `ui-token-rotation` | **Date**: 2026-05-19 | **Spec**: [spec.md](spec.md)

---

## Summary

The vJailbreak UI embeds the Kubernetes ServiceAccount token in the browser at page load (`startup.sh` reads the token and substitutes it into `index.html`). The browser then sends this token on every API call. The token has no expiration. This plan replaces that mechanism with a server-side proxy: the UI container's OpenResty/Nginx reads the token from a projected volume and injects it as an outgoing `Authorization` header — the browser never receives or transmits the token.

A prerequisite routing change is required: `vjailbreak-api-ingress` currently routes `/api.*` and `/apis.*` directly to the Kubernetes API server, bypassing the UI container. Deleting this Ingress causes those paths to fall through to the UI service, enabling the Nginx proxy.

## Phase 1: Kubernetes Deployment Changes

### T010 — Add projected ServiceAccount token volume to UI Deployment

**File**: `ui/deploy/ui.yaml`  
**Changes**:

1. On the Deployment spec, add `automountServiceAccountToken: false` (prevents co-existing long-lived token)
2. Add projected volume `sa-token`:

   ```yaml
   - name: sa-token
     projected:
       sources:
       - serviceAccountToken:
           expirationSeconds: 86400   # 24h default; operator-configurable
           path: token
       - configMap:
           name: kube-root-ca.crt
           items:
           - key: ca.crt
             path: ca.crt
       - downwardAPI:
           items:
           - path: namespace
             fieldRef:
               apiVersion: v1
               fieldPath: metadata.namespace
   ```

3. Add volumeMount to the container:

   ```yaml
   - name: sa-token
     mountPath: /var/run/secrets/kubernetes.io/serviceaccount
     readOnly: true
   ```

**Note**: `deploy/07ui-deployment.yaml` and `deploy/installer.yaml` are generated from this file by the pre-commit hook. Do not edit them directly.

### T011 — Delete `vjailbreak-api-ingress` from `ui/deploy/ui.yaml`

**File**: `ui/deploy/ui.yaml`  
**Change**: Remove the entire `vjailbreak-api-ingress` Ingress resource (lines ~82–111 in the current file).

**Effect**: After deletion, browser requests to `/api/v1/...` and `/apis/...` are caught by `vjailbreak-ui-ingress` (path `/`) and routed to `vjailbreak-ui-service:80` → UI container Nginx. The Nginx proxy added in T012 then handles them.

**Verify**: No other ingress in any namespace provides a more-specific match for `/api/` or `/apis/`. Grafana ingress (`/grafana`) and vpwned ingress (`/dev-api/sdk/`) are unaffected.

## Phase 2: Nginx Proxy Configuration

### T013 — Add server-side token injection to `ui/default.conf`

**File**: `ui/default.conf`  
**Change**: In the `http { server { ... } }` block, add:

1. Variable declarations (before `location` blocks):

   ```nginx
   set $kube_api https://kubernetes.default.svc;

   set_by_lua_block $sa_token {
       local f, err = io.open("/var/run/secrets/kubernetes.io/serviceaccount/token", "r")
       if not f then
           ngx.log(ngx.ERR, "Failed to open SA token: ", err)
           return ""
       end
       local token = f:read("*a")
       f:close()
       return (token:gsub("%s+$", ""))
   }

   proxy_ssl_trusted_certificate /var/run/secrets/kubernetes.io/serviceaccount/ca.crt;
   proxy_ssl_verify on;
   proxy_ssl_server_name on;
   proxy_ssl_name kubernetes.default.svc;
   proxy_set_header Host kubernetes.default.svc;
   proxy_set_header Authorization "Bearer $sa_token";
   proxy_http_version 1.1;
   proxy_set_header Upgrade $http_upgrade;
   proxy_set_header Connection $connection_upgrade;
   ```

2. Add location blocks (before the `location /` block):

   ```nginx
   location ^~ /api/ {
       proxy_buffering off;
       proxy_pass $kube_api;
   }

   location ^~ /apis/ {
       proxy_buffering off;
       proxy_pass $kube_api;
   }
   ```

**Note**: `proxy_set_header Authorization` applies to all `proxy_pass` locations unless overridden. The existing `location /` block (static files) does not use `proxy_pass`, so it is unaffected.

**WebSocket support**: The `proxy_http_version 1.1` + `Upgrade`/`Connection` headers are already set (for pod log streaming). Confirm they apply to the new location blocks.

## Phase 5: k3s Configuration

### T020 — Disable k3s token extension flag

**File**: `image_builder/configs/k3s.config.yaml`  
**Change**: Ensure the following entries are present:

```yaml
service-account-extend-token-expiration: false
secrets-encryption: true
```

`service-account-extend-token-expiration: false` prevents k3s from silently extending projected token TTLs beyond the configured `expirationSeconds`. Without this flag, the 24h TTL may be extended indefinitely, defeating the purpose.
