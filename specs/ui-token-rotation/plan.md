# Implementation Plan: UI Service Account Token Security

**Branch**: `ui-token-rotation` | **Date**: 2026-05-04 | **Spec**: [spec.md](spec.md)  
**Updated**: 2026-05-04 — revised to adopt Nginx server-side proxy approach (branch `private/main/omkard/ui-hide-rotate-token`)

## Summary

Two-part fix: (1) move all Kubernetes API calls behind a server-side Nginx proxy that injects the SA token — token never reaches the browser; (2) configure the SA token as a projected volume with `expirationSeconds` so kubelet rotates it automatically. No frontend token handling, no TokenRequest API calls, no axios interceptor changes.

The Nginx proxy changes, k3s config, and frontend token removal are **already implemented** on branch `private/main/omkard/ui-hide-rotate-token`. The remaining work is adding the projected token volume to the Deployment.

## Technical Context

**Language/Version**: Nginx/Lua (OpenResty) for proxy; Kubernetes YAML for infra; k3s config YAML  
**Primary Dependencies**: Kubernetes projected ServiceAccount tokens (native kubelet rotation), OpenResty Lua (existing), k3s `service-account-extend-token-expiration` flag  
**Storage**: Token file on disk at `/var/run/secrets/kubernetes.io/serviceaccount/token` (managed by kubelet)  
**Testing**: Manual k8s manifest validation; browser inspection (no token in page source/network)  
**Target Platform**: k3s on Linux (vJailbreak appliance)  
**Project Type**: Security hardening — server-side proxy + bounded token lifetime  
**Performance Goals**: Per-request Lua file read adds <1ms latency; acceptable  
**Constraints**: No frontend changes beyond removing token injection; no new services or sidecars  
**Scale/Scope**: Single UI pod; all browser sessions benefit equally

## Implementation Tasks

### Task 1 — Infra: Projected Token Volume (deploy/07ui-deployment.yaml) ← REMAINING WORK

**Goal**: Replace the automounted (never-expiring) SA token with a projected token having a bounded TTL. Kubelet rotates the file on disk automatically; Nginx picks up the new token on the next request with no restart.

**Why this is needed**: The Nginx proxy reads `/var/run/secrets/kubernetes.io/serviceaccount/token` on each request (already implemented). Without this task, that file still contains a never-expiring token — Nginx serves a rotating file, but the file never actually rotates.

**Changes to `deploy/07ui-deployment.yaml`**:

1. Add `automountServiceAccountToken: false` to the pod spec — disables the long-lived automounted token.

2. Add projected volume `sa-token` replacing all three files the automount provides:

```yaml
spec:
  automountServiceAccountToken: false
  volumes:
  - name: nginx-shadow-htpasswd
    hostPath:
      path: /etc/htpasswd
      type: FileOrCreate
  - name: pf9-logs
    hostPath:
      path: /var/log/pf9
      type: DirectoryOrCreate
  - name: sa-token
    projected:
      sources:
      - serviceAccountToken:
          expirationSeconds: 86400
          path: token
      - configMap:
          name: kube-root-ca.crt
          items:
          - key: ca.crt
            path: ca.crt
      - downwardAPI:
          items:
          - path: namespace
            fieldRef:
              apiVersion: v1
              fieldPath: metadata.namespace
  containers:
  - name: vjailbreak-ui-container
    volumeMounts:
    - name: nginx-shadow-htpasswd
      mountPath: /etc/nginx/shadow
    - name: pf9-logs
      mountPath: /var/log/pf9
      readOnly: true
    - name: sa-token
      mountPath: /var/run/secrets/kubernetes.io/serviceaccount
      readOnly: true
```

**To change TTL**: Update `expirationSeconds` in this file and roll the Deployment. Pod restart is required and expected.

**Minimum TTL**: 600 seconds (enforced by Kubernetes API server).  
**Default TTL**: 86400 seconds (24 hours).

## Already Implemented (branch `private/main/omkard/ui-hide-rotate-token`)

These changes are complete and should be cherry-picked or merged as the base for Task 1:

### k3s Config (`image_builder/configs/k3s.config.yaml`)

```yaml
secrets-encryption: true
kube-apiserver-arg:
  - "service-account-issuer=https://kubernetes.default.svc"
  - "service-account-signing-key-file=/var/lib/rancher/k3s/server/tls/service.key"
  - "service-account-extend-token-expiration=false"
```

`service-account-extend-token-expiration=false` is the critical flag — prevents k3s from silently extending projected token lifetimes beyond `expirationSeconds`.

### Nginx Proxy (`ui/default.conf`)

Lua block reads token from disk on every request:

```lua
set_by_lua_block $sa_token {
    local f = io.open("/var/run/secrets/kubernetes.io/serviceaccount/token", "r")
    if not f then return "" end
    local token = f:read("*a")
    f:close()
    return (token:gsub("%s+$", ""))
}
```

Proxy locations inject the token server-side:

```nginx
proxy_set_header Authorization "Bearer $sa_token";
proxy_ssl_trusted_certificate /var/run/secrets/kubernetes.io/serviceaccount/ca.crt;
proxy_ssl_verify on;

location ^~ /api/ { proxy_buffering off; proxy_pass $kube_api; }
location ^~ /apis/ { proxy_buffering off; proxy_pass $kube_api; }
location ^~ /sdk/ { proxy_buffering off; proxy_pass $kube_api; }
```

WebSocket support included (`Upgrade`/`Connection` headers set).

### startup.sh

Token injection into `index.html` removed entirely — `startup.sh` now only starts OpenResty.

### axios.ts + pods.ts

`Authorization: Bearer ${VITE_API_TOKEN}` removed from both files. Frontend sends no auth header — proxy handles it.

## Error Handling

| Scenario | Behaviour |
|----------|-----------|
| Lua fails to open token file | Nginx logs ERR; request proceeds with empty Authorization header → k8s API returns 401; UI shows error |
| Token expires before kubelet rotates it | Short gap possible on very short TTLs; kubelet rotates at ~80% of TTL so this should not occur under normal conditions |
| `expirationSeconds` below 600s | k8s API server rejects pod spec; pod fails to start — admin must fix Deployment |
| Admin changes `expirationSeconds` | Rolling restart; fresh projected token at new TTL; Nginx immediately uses new file on next request |
| Pod restart during browser session | Nginx restarts; in-flight requests may fail momentarily; browser retries recover automatically |

## Rollback Plan

Revert `07ui-deployment.yaml` (remove projected volume, remove `automountServiceAccountToken: false`). Revert `default.conf` to return 404 on `/api/` and `/apis/`. Restore token injection in `startup.sh`. Restore `Authorization` header in `axios.ts` and `pods.ts`. The long-lived automounted token returns.
