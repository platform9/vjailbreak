## Phase 4: User Story 2 вҖ” Automatic Token Renewal (Priority: P2)

**Goal**: Token renews automatically via kubelet projected volume. No pod restart needed ever.

**Independent Test**: `kubectl exec <ui-pod> -- cat /var/run/secrets/kubernetes.io/serviceaccount/token | cut -d'.' -f2 | base64 -d` вҶ’ verify `exp` claim is set and `exp - iat вүӨ 86400`. For TTL test: set `expirationSeconds: 600`, wait 8+ minutes, verify token file content changes while API calls continue to succeed.

**Note**: Kubelet handles renewal at ~80% of TTL (вүҲ 19.2h for 24h default). No code needed beyond T002. This task locks k3s to respect the configured TTL.

- [ ] T015 [US2] Add `service-account-extend-token-expiration: false` and `secrets-encryption: true` to `image_builder/configs/k3s.config.yaml` вҖ” prevents k3s from silently extending projected token TTLs beyond the configured `expirationSeconds`

**Checkpoint**: Token `exp` claim matches configured TTL. k3s will not extend it indefinitely.
