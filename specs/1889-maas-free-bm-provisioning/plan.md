## Summary

Two-phase feature. Phase 1 adds a per-ESXi accordion UI to the existing ClusterConversionsPage,
creates a VMwareCluster controller to track per-host VM counts, and decouples ESXIMigration from
ClusterMigration by making RollingMigrationPlanRef optional. Phase 2 adds Ironic and Direct IPMI
providers to the existing BMCProvider abstraction, removing the hard MAAS dependency.

---

**IronicProvider** (`pkg/vpwned/sdk/providers/ironic/ironic.go`):

- Use `gophercloud` (OpenStack Go SDK) for Ironic REST API
  - **First**: verify or add `gophercloud` to `pkg/vpwned/go.mod`:
    `cd pkg/vpwned && go get github.com/gophercloud/gophercloud`
  - Consult [Ironic API docs](https://docs.openstack.org/ironic/latest/api/) before implementing
- Method mapping:
  - `Connect()` → authenticate to Keystone, get Ironic client
  - `ListResources()` → `GET /v1/nodes`; filter by instance_info or chassis UUID
  - `GetResourceInfo()` → `GET /v1/nodes/{node_ident}`
  - `DeployMachine()` → set `instance_info` (image URL, cloud-init), `PUT /v1/nodes/{}/provision` → `active`
  - `ReclaimBM()` → `PUT /v1/nodes/{}/provision` → `available` (triggers cleaning)
  - `SetBM2PXEBoot()` → `PATCH /v1/nodes/{}` set `boot_interface: pxe`
  - `StartBM()` / `StopBM()` → `PUT /v1/nodes/{}/states/power`
  - Unsupported methods (MAAS-specific) → return `ErrNotSupported`

**Unit tests**: Mock gophercloud HTTP client; test each method against expected API calls.

**Provider registration**: `IronicProvider.init()` self-registers via `providers.RegisterProvider("ironic", ...)` (already in stub). Add blank import `_ "github.com/platform9/vjailbreak/pkg/vpwned/sdk/providers/ironic"` to `k8s/migration/pkg/utils/bmprovisionerutils.go` to trigger it.

### Milestone 7 — Direct IPMI/Redfish Provider

*Commit: `feat(providers): add direct IPMI/Redfish BMC provider`*

**IPMIProvider** (`pkg/vpwned/sdk/providers/ipmi/ipmi.go`):

Reuse IPMI calls already present in `maas.go`:

- `ChassisControlPowerUp` → `StartBM()`
- `ChassisControlPowerDown` → `StopBM()`
- `SetBM2PXEBoot()` → chassis bootdev pxe + power cycle (extract `ChassisControlPowerDown`, `ChassisControlPowerUp`, and PXE bootdev sequence from `maas.go`)

Redfish path (when `IPMIConfig.UseRedfish = true`):

- Use `gofish` library (`github.com/stmcginnis/gofish`) for Redfish REST calls
- Power management: `POST /redfish/v1/Systems/{id}/Actions/ComputerSystem.Reset`
- Boot override: `PATCH /redfish/v1/Systems/{id}` BootSourceOverride

`DeployMachine()`:

- Serve cloud-init `user-data` from vJailbreak VM's local HTTP endpoint
  (vJailbreak VM already has network access; add simple HTTP handler or reuse existing one)
- After PXE boot, machine fetches cloud-init and self-enrolls in PCD

`ListResources()`:

- Returns single-element list using `BMCAddress` from `IPMIConfig`
  (no auto-discovery; user configures one BMC per `BMConfig`)

Unsupported: `ListBootSource`, MAAS-specific methods → `ErrNotSupported`

**Unit tests**: Mock IPMI/Redfish calls; test boot device set, power cycle sequence.
