How does the user reach an ESXi host page?

This determines the top-level navigation structure for the redesigned cluster conversion flow.

A

Cluster → ESXi Hosts (drill-down)

User picks a VMware cluster first. The cluster page lists all its ESXi hosts with capacity bars. Clicking a host opens the per-ESXi detail page.

Cluster Conversions › my-cluster › esxi-01
B

Flat ESXi host list (no cluster grouping)

One page shows all ESXi hosts across all clusters, sorted by eligibility (empty hosts first). Clicking a host opens the per-ESXi detail page.

Host Migrations › esxi-01
C

Cluster page with expandable ESXi rows (no separate drill-down)

Cluster page shows collapsible rows — expand an ESXi row in-place to see its VMs, capacity, and actions. No separate page per host.

Cluster Conversions › my-cluster [▼ esxi-01 expanded]
